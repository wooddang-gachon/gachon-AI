# 타이타닉 생존자 예측 및 피처 선택 분석 (Titanic Survival Classification)

이 프로젝트는 타이타닉 호의 승객 데이터를 활용하여 생존 여부를 예측하는 머신러닝 분류 모델을 구현한 프로젝트입니다. 특히 **전체 데이터를 사용했을 때**와 **상관관계 분석을 통해 주요 피처만 선택했을 때**의 모델 성능을 비교 분석했습니다.

## 문제점

현재 데이터셋의 절대적인 n수가 작아 정상적인 데이터가 나오지 않는 것 같다. 실제로 히트맵을 기준으로 연관성이 높은 데이터만 뽑았음에도 불구하고 결과값이 낮게 나오는 경우가 발생하고 있다. 이는 다른 데이터(N수가 많은 걸로)로 비슷한 테스트를 이후 진행해보아야 함을 시사한다.

## 1. 프로젝트 개요

- **목표**: 승객의 데이터를 바탕으로 생존 여부(Survived) 예측
- **주요 파일**:
  - `TitanicClassifier.ipynb`: 불필요한 일부 식별용 데이터(이름, 티켓 등)만 제거하고, 대부분의 피처를 사용하여 모델 학습
  - `TitanicClassifierFeatureSelection.ipynb`: 상관관계 분석(Heatmap)을 통해 생존율과 연관성이 높은 주요 피처 3개(`Pclass`, `Sex`, `Age`)만 선별하여 모델 학습

## 2. 데이터셋 및 전처리

**사용 데이터**: Kaggle Titanic Dataset (`titanic.csv`)

### 컬럼(피처) 설명

- `PassengerId`: 승객의 고유 번호
- `Survived`: 생존 여부 (0 = 사망, 1 = 생존) **[예측 대상]**
- `Pclass`: 객실 등급 (1 = 1등석, 2 = 2등석, 3 = 3등석)
- `Name`: 승객 이름
- `Sex`: 성별 (male, female)
- `Age`: 나이
- `SibSp`: 함께 탑승한 형제자매, 배우자의 수
- `Parch`: 함께 탑승한 부모, 자녀의 수
- `Ticket`: 티켓 번호
- `Fare`: 탑승 요금
- `Cabin`: 객실 번호
- `Embarked`: 탑승 항구 (C = Cherbourg, Q = Queenstown, S = Southampton)

### 전처리 과정

- **결측치 처리**:
  - `Age` (나이): 데이터의 중앙값(median)으로 대체하여 이상치의 영향을 최소화
  - `Embarked` (탑승 항구): 최빈값(mode)으로 빈 공간 대체
- **데이터 인코딩**: 머신러닝 모델이 학습할 수 있도록 문자열을 숫자로 변환
  - `Sex`: 남성(male)은 0, 여성(female)은 1로 매핑
  - `Embarked`: LabelEncoder를 사용하여 각 항구 알파벳을 수치형 범주로 변환

## 3. 분석 방법 및 피처 선택 (Feature Selection)

### 히트맵(Heatmap)을 사용한 이유

수많은 변수들 사이의 복잡한 관계를 한눈에 파악하고, 우리의 예측 대상인 **'생존 여부(Survived)'와 직결되는 핵심 변수를 과학적이고 객관적으로 선별하기 위해** 상관계수 히트맵 시각화를 진행했습니다.

### 주요 피처 3가지의 특징과 의미

상관관계 분석 결과, 아래 3개의 피처가 생존율과 가장 유의미한 관계를 맺고 있음을 확인했습니다.

1. **Sex (성별)**
   - 생존율과 가장 강한 상관관계를 보였습니다. "여성과 아이 먼저"라는 당시 해상 사고의 암묵적 구조 원칙이 데이터에 명확히 반영되어, 여성(1)의 생존율이 남성(0)보다 월등히 높았음을 알 수 있습니다.
2. **Pclass (객실 등급)**
   - 뚜렷한 음(-)의 상관관계를 보였습니다. 1등석(1)에 가까울수록 생존 확률이 높고, 3등석(3)으로 숫자가 커질수록 생존 확률이 낮아졌음을 의미하는 결정적 지표입니다.
3. **Age (나이)**
   - 나이 자체의 선형적 상관계수 절대값은 크지 않을 수 있으나, 어린아이들의 높은 생존율 등 성별 및 객실 등급과 결합하였을 때 생존 여부를 유추하는 데 있어 모델에 필수적인 정보를 제공하는 핵심 피처로 판단하여 포함했습니다.

## 4. 모델 성능 비교 (테스트셋 25% 기준)

다양한 분류 알고리즘을 적용하여 두 가지 데이터셋(전체 피처 vs 선택된 피처)의 예측 정확도(Accuracy)를 비교했습니다.

| 모델 (Model)             | 전체 피처 사용 | 주요 피처(3개) 선택 |
| :----------------------- | :------------- | :------------------ |
| **Random Forest**        | 83.8%          | 80.2%               |
| **MLP (Neural Network)** | 81.1%          | 78.9%               |
| **Logistic Regression**  | 79.3%          | 79.3%               |
| **Decision Tree**        | 78.0%          | 78.0%               |
| **SVM (SVC)**            | 72.1%          | 64.1%               |

_(KNN 모델의 경우, K값 변화에 따른 성능 그래프를 노트북 파일 내에서 시각화하여 최적점을 도출했습니다.)_

## 5. 결론 및 시사점

본 프로젝트는 단순히 정확도를 높이는 것을 넘어, **데이터의 차원 축소(피처 선택)가 모델에 미치는 효과**를 입증하는 데 의의가 있습니다.

1. **가장 우수한 모델**: 두 가지 실험 모두에서 **Random Forest** 앙상블 모델이 가장 높은 정확도를 기록하여, 본 데이터셋의 비선형적인 관계를 가장 잘 학습함을 확인했습니다.
2. **피처 선택(Feature Selection)의 4가지 주요 효과**:
   - **핵심 정보의 증명**: 10개가 넘는 피처를 모두 사용했을 때(약 83.8%)와 비교해, 단 3개(`Pclass`, `Sex`, `Age`)의 피처만 사용했을 때도 **80.2%** 라는 방어력 높은 성능을 기록했습니다. 이는 전체 데이터의 예측 신호(Signal) 대부분이 이 3가지 피처에 몰려 있음을 의미합니다.
   - **다중공선성(Multicollinearity) 완화**: `Fare`(요금)와 `Pclass`(객실 등급)처럼 서로 상관성이 높은 변수들을 동시에 넣으면 모델이 불안정해질 수 있습니다. 피처 선택을 통해 이러한 중복 피처를 제거하여 모델의 안정성을 높였습니다.
   - **과적합(Overfitting) 방지 및 연산 효율성**: 불필요한 노이즈 데이터를 걷어냄으로써 새로운 데이터에 대한 일반화(Generalization) 성능을 높일 수 있는 기반을 마련했고, 모델 학습 속도와 연산 비용을 획기적으로 줄였습니다.
   - **결과 해석의 용이성(Interpretability)**: 수많은 변수가 섞인 블랙박스 모델보다, "성별, 객실 등급, 나이"라는 직관적이고 상식적인 3가지 조건만으로 생존율을 설명할 수 있어 분석 결과의 설득력이 매우 높아졌습니다.
